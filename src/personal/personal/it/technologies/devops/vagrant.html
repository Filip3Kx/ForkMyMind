Vagrant

Basic Use

Po stworzeniu folderu, który jest lokalizacją i nazwą VM'ki:

#setup
vagrant init #nazwa obrazu

#on
vagrant up

#remote access 
vagrant ssh

#off
vagrant halt

#rm
vagrant destroy

#reset i przemielenie Vagrantfile'a jeszcze raz
vagrant reload

Vagrant file

Status wszystkich VM'ek

vagrant global-status

Ustawienia 

config.vm.box = nazwa obrazu

config.vm.network "public_network" #bridged
config.vm.network "private_network", ip: "192.168.0.1"

config.vm.provider:
	vb.gui = true
	vb.memory = 1024
	vb.cpus = 1

Dzielone foldery

Bazowo /vagrant jest dzielony z rootem vm'ki na hoście (tam gdzie jest Vagrantfile)

Dzielenie folderu (podwójne \\ w ścieżce windowsowej)

config.vm.synced_folder "local_path", "remote_path"

Provisioning (odpalanie komend przy starcie)

Provisioning = bootstraping

Provisioners

  • Ansible
  • Chef
  • Docker
  • Puppet
  • Salt

Konfiguracja

config.vm.provision "shell", inline: <<-SHELL
	apt-get update
	apt-get install -y apache2
SHELL

Przy reloadzie trzeba dodać przełącznik bo komendy wykonują się tylko raz

vagrant reload --provision

Multimachine

Po stworzeniu swojego pliku Vagrantfile można stworzyć w nim konfiguracje dla wielu różnych vm'ek

Vagrant.configure("2") do |config|
  config.vm.provision "shell", inline: "echo Hello"

  config.vm.define "web" do |web|
    web.vm.box = "apache"
  end

  config.vm.define "db" do |db|
    db.vm.box = "mysql"
  end
end

W tym przykładzie mamy 2 maszyny db i web do których potem się odnosimy jako obiektów

Note

bardziej zaawansowany plik robiony w projekcie Vprofile
