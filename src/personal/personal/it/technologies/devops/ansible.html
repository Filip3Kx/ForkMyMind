Install

Wymagane są python i SSH lub winrm i powershell dla Windowsa.

sudo apt update
sudo apt install software-properties-common
sudo apt-add-repository --yes --update ppa:ansible/ansible
sudo apt install ansible

Creating inventory

the default inventory of ansible is in /etc/ansible/hosts
Here is a sample one to show how to use it

all:
  children:
    d-r01:
      hosts:
        10.237.147.121:
        10.237.146.161:
        10.237.146.113:
        10.237.146.209:
        10.237.146.211:
        10.237.146.213:
        10.237.146.215:
        10.237.146.217:
        10.237.146.219:
  vars:
    ansible_user: test
    ansible_ssh_pass: test
    ansible_remote_tmp: /tmp
Caution

If you are using credentials instead of keys you need to turn of key checking in /etc/ansible/ansible.cfg. To do that paste this

[defaults]
host_key_checking = False

Yaml basics

Tworzenie zmiennych lub parametrów dla ansible w formie pojedyńczej zmiennej lub listy

name: cos1

names:
  - cos1
  - cos2
  - cos3

names: [cos1, cos2, cos3]

Zagniezdzenie zmiennej w stringu

name_output: "{{name}}"

Creating a playbook

Playbooks are yaml files that are run with ansible-playbook command

Here is a sample playbook that will get kernel and OS versions from all hosts

- name: Retrieve Kernel and OS Version
  hosts: all
  gather_facts: yes

  tasks:
    - name: Get Kernel Version
      command: uname -r
      register: kernel_output
      changed_when: false

    - name: Get OS Version
      command: lsb_release -d -s
      register: os_output
      changed_when: false

    - name: Display Kernel and OS Version
      debug:
        msg:
          - "Kernel Version: {{ kernel_output.stdout }}"
          - "OS Version: {{ os_output.stdout }}"

To run a playbook you can use this command that will also use -l to limit the hosts: all statement in our code

ansible-playbook -l d-r01 version.yaml
